## Webhooks Overview

Webhooks in Space Energy Documentation enable real-time notifications and automated workflows by sending HTTP requests to specified URLs when certain events occur in your documentation spaces. This powerful feature allows you to integrate documentation updates with other tools and services.

## Security and Authentication

Secure your webhooks to prevent unauthorized access and ensure data integrity.

<Steps>
  <Step title="Use HTTPS" icon="lock">
    Always use HTTPS URLs for webhook endpoints to encrypt data in transit.
  </Step>
  <Step title="Verify Signatures" icon="key">
    Implement signature verification to confirm webhook authenticity.
  </Step>
  <Step title="Validate Payloads" icon="check">
    Check that received payloads match expected structure and content.
  </Step>
</Steps>

<CodeGroup tabs="JavaScript,Python">
  ```javascript
const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');
  return signature === `sha256=${expectedSignature}`;
}
```
  ```python
import hmac
import hashlib
import json

def verify_webhook(payload, signature, secret):
    expected = hmac.new(
        secret.encode(),
        json.dumps(payload, sort_keys=True).encode(),
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(signature, f'sha256={expected}')
```
</CodeGroup>

# Test webhook endpoint

    'event': 'document.updated',
    'timestamp': datetime.now().isoformat(),
    'space': {'id': 'test-space', 'name': 'Test Space'},
    'document': {'id': 'test-doc', 'title': 'Test Document'}
